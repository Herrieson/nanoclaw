Use this skill when the task asks for a tutorial brief, workspace handoff, or compact project summary.

Guidelines:

- Read memory and the relevant workspace files before drafting conclusions.
- Keep the output short, factual, and easy to scan.
- Mention preferences only when memory supports them.
- Prefer `read`, `write`, `edit`, and `apply_patch` over `exec` for file-only work.
- End with one concrete next step.
