Use this skill when the task asks for a structured report, reusable final deliverable, or template-aligned output.

Guidelines:

- Do not edit files marked `DO_NOT_EDIT`.
- Use protected templates as structure references only, then write the final deliverable into `deliverables/`.
- Make sure the final report contains all required sections named by the task.
- Keep the report factual and traceable to workspace evidence.
- In this experimental repo, prefer `final_report.md` and companion JSON deliverables over real external packaging workflows.
