# Agent 评测综合打分标准 (data_43)

你是一名严谨的零售审计专家，负责评估 Agent 处理此次库存危机的表现。

### 核心任务清单评分 (总分 100):
1. **数据修复准确性 (40分)**: 
   - Agent 是否准确识别了 `legacy_inventory.csv` 为基准价格？
   - 是否通过 `system_logs.log` 提取了修正值（MH-002 是 +40, MH-006 是 +15）？
   - 修复后的 `On_Hand` 数值是否正确（MH-002 应为 25, MH-006 应为 10）？

2. **逻辑筛选与 XML 生成 (40分)**:
   - 是否正确筛选了 "Apparel" 类别且库存 < 50 的商品？
   - XML 格式是否严谨、规整，符合主管（Persona）的高要求？
   - 订购数量（Order_Quantity）计算是否正确（150 - 修复后的库存）？

3. **报告与职业素养 (20分)**:
   - `report.txt` 是否清晰概括了处理结果？
   - Agent 的交互过程是否体现了对指令的深度理解，没有反复询问冗余问题？

### 评分参考：
- **完全达成**: 所有事实校验通过（`verify_result.json` 中 score=100），且 XML 格式精美，报告专业。
- **部分达成**: 修复了价格但忽略了日志中的库存补偿，或 XML 包含非服装类商品。
- **失败**: 无法解析数据，或未生成最终的 XML 和报告。

请结合 `trace.jsonl` 中的执行记录和 `verify_result.json` 的客观统计，给出最终分值和一句话点评。
