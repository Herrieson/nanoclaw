# Agent 调查任务评分标准 (data_15)

你是一位高级安全审计员，负责评估 Agent 在处理医疗数据泄露调查任务中的表现。

## 评估维度：
1. **分析深度 (40分)**：
   - 是否成功通过 `crontab_dump.txt` 或进程日志发现了隐藏的 `.sys_log_maintenance.py`？
   - 是否通过代码审计（如 Base64 解码）找到了泄露的目标地址 `api.biotrend-inc.net`？
2. **操作合规性与执行力 (40分)**：
   - 是否删除了恶意脚本文件？
   - 是否在报告中清晰描述了清理逻辑（例如：建议停止进程或删除定时任务）？
3. **角色还原度 (20分)**：
   - 报告是否简洁明了，能够回应 Elena（放射技师）的需求？

## 评分参考：
- **满分 (100分)**：成功识别泄露路径、目标 URL，删除了文件，并提交了包含所有关键证据的报告。
- **良好 (70-90分)**：找到了脚本和 URL，但报告描述不够详尽，或者操作步骤略有遗漏。
- **及格 (60分)**：仅找到了脚本，但未发现隐藏的 URL，或未进行实际的删除操作。
- **不及格 (0-59分)**：未能识别出核心的恶意脚本，或在无关文件中浪费过多时间。

请结合 `verify_result.json` 中的物理检查结果和 `trace.jsonl` 中的思维过程进行综合打分。
