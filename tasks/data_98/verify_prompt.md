# 裁判指导

本任务要求 Agent 扮演一个缺乏耐心、脾气暴躁的起重机操作员的助手，解析其杂乱的木工裁切清单，计算需要的 2x4 和 4x4 木材总长度（以英尺为单位），同时必须敏锐地在日志文件中发现废弃的图纸要求并做出相应的排除和追加计算。

请根据提供的 `verify_result.json`（事实检验结果）以及 Agent 的执行轨迹进行打分。

## 验证项说明
- `file_exists`: 是否成功在根目录下创建了 `lumber_run.txt` 文件。
- `2x4_correct`: 2x4 的长度是否计算正确（正确值：77英尺。推导过程：野餐桌 65英尺 + 边桌 12英尺 = 77英尺）。
- `4x4_correct`: 4x4 的长度是否计算正确（正确值：18英尺。推导过程：野餐桌 12英尺 + 边桌 6英尺 = 18英尺）。

## 评分标准 (0-100)
- **100分**: 所有三项皆为 `true`。Agent 完美地找到了所有文件，忽略了无效的椅子图纸，正确转换了英寸到英尺并得出了绝对正确的长度总和。
- **70分**: 创建了文件，但其中有一个尺寸计算错误（通常是因为没有看到 `notes.log` 中忽略椅子的要求，或者英寸换算英尺发生计算错误）。
- **40分**: 创建了文件，但两个尺寸都计算错误（完全没有理解指令或数学逻辑崩坏）。
- **0分**: 没有找到目标文件夹，或完全没有创建 `lumber_run.txt` 文件。
