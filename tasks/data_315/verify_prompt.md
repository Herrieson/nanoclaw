# Agent 评测任务评分标准 (data_315)

你将根据 Agent 的执行日志 (`trace.jsonl`) 和物理状态校验结果 (`verify_result.json`) 进行评分。

### 核心任务指标 (总分 100 分)
1. **废油异常检测 (40分)**：
   - 是否准确识别出 2023-10-04 和 2023-10-06 的废油记录存在显著差异？
   - 是否正确读取了 SQLite 数据库和传感器日志并进行了跨比对？
   - 分数分配：发现逻辑正确得 20 分，结果数值准确得 20 分。

2. **采购合规性分析 (40分)**：
   - 是否关联了供应商 JSON、采购记录 JSON 以及地理位置/认证状态？
   - 计算的比例是否在 20.7% 左右？
   - 分数分配：多文件关联逻辑正确得 20 分，最终计算比例正确得 20 分。

3. **系统操作与规范性 (20分)**：
   - 是否独立编写了处理脚本而不是硬编码？
   - 是否按照 Astrid 要求的极简风格生成了 JSON 报告？
   - 如果 Agent 尝试猜测数据而非读取文件，此处扣除 20 分。

### 评分参考
- **完全达成**：`verify_result.json` 中所有项为 true，且 trace 显示其逻辑严密（查询了笔记，通过 SQL 聚合数据）。
- **部分达成**：仅完成了废油或采购中的一项，或计算结果有小偏差。
- **失败**：未能生成报告，或报告中的关键日期和比例完全错误。

请根据 `verify_result.json` 的客观结果和 `trace.jsonl` 中的思维过程给出最终得分。
