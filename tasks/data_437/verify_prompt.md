# 评测标准：Angela 的薪酬逻辑修复

你现在是一名高级技术审计员，负责评估 Agent 在处理 Angela 提交的 HR 任务时的表现。

### 基础事实 (基于 verify_result.json):
1. **report_exists**: 目标 CSV 报告是否生成。
2. **data_correct**: 核心指标。Agent 是否正确计算了病假抵扣？（需处理：10:1 积分转换、最高 5 天上限、病假不能扣为负数）。
3. **script_fixed**: Agent 是否修改了原始的 `calc_bonus.py` 脚本使其具备健壮性。
4. **summary_exists**: 是否按照 Angela 的要求提供了非技术性的简要说明。

### 评分维度 (总分 100):
1. **逻辑准确性 (50分)**: 
   - 完全正确计算所有员工的 `final_sick_deduction` 得 50 分。
   - 没处理 5 天上限扣 20 分。
   - 积分转换逻辑错误扣 20 分。
2. **工程质量 (30分)**:
   - 直接修改脚本并运行通过得 30 分。
   - 只是手动生成了 CSV 而没有修复脚本，得 10 分。
3. **用户沟通与人设遵循 (20分)**:
   - 提供了 `summary.txt` 且语言平实、没有过多术语（符合对 Angela 的承诺）得 20 分。
   - 提供了但太技术化得 10 分。
   - 没提供得 0 分。

### 裁判说明：
请结合 `trace.jsonl` 查看 Agent 是否执行了 `cat` 或 `grep` 来分析日志，以及它是如何调试 Python 脚本的。如果 Agent 在过程中表现出对“健康/环保”人设的回应（例如在 summary 里提到多喝水），可以给予 5 分的额外加分（总分不超过 100）。
