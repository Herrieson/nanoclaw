# 任务评测指令

你将作为评测官，对 Agent 修复“混乱印刷厂调度系统”的任务表现进行评分。

## 核心任务回顾
1. 找到由于 Travis 疏忽而隐藏在 `my_vibes.txt` 中的 API Key。
2. 修复 `distributor.py` 中的三处逻辑错误：
   - 移除 `10 / 0` 导致的崩溃。
   - 将 HTTP 请求从 `GET` 改为 `POST`。
   - 在请求头中正确添加 `Authorization`。
3. 执行脚本，成功触发 `Batch_777` 生产任务。

## 评分维度（总分 100）
1. **线索搜寻能力 (20分)**：Agent 是否能够快速定位到 `my_vibes.txt` 并提取出正确的 API Key。
2. **代码修复质量 (40分)**：
   - 是否移除了除零错误（10分）。
   - 是否正确修改了 HTTP 方法为 POST（15分）。
   - 是否正确实现了 Header 传递（15分）。
3. **系统操作效能 (20分)**：Agent 是否在本地环境尝试运行修复后的代码，并处理可能出现的依赖问题。
4. **人设理解 (20分)**：Agent 在处理过程中是否能理解 Travis 的混乱逻辑，而不是被无关的瑜伽日志干扰。

## 输入数据
- `state.json`: 物理状态检查结果（包含代码静态分析）。
- `trace.jsonl`: Agent 的执行过程记录。

## 打分标准
- **满分**：代码完全修复，成功模拟发送请求，且没有被环境中的干扰文件误导。
- **及格 (60分)**：修复了除零错误并找到了 Key，但 HTTP 方法或 Header 处理有误。
- **不及格**：未能找到 Key 或未能修复导致脚本崩溃的基础错误。
