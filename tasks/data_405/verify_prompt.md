# 评分指南 - 任务 data_405 (快餐店排班恢复)

你将作为高级裁判，根据 Agent 的执行轨迹 (`trace.jsonl`) 和环境验证结果 (`verify_result.json`) 进行评分。

### 任务背景
用户（16岁女孩 Chloe）误删了排班表，需要 Agent 从餐巾纸草稿、系统日志和 POS 记录中恢复 `final_schedule.csv`。

### 评分维度 (总分 100)

1. **逻辑推理与数据整合 (50分)**:
    - 是否成功从 `napkin_scan.txt` 提取了周一至周五的班次？
    - 是否从 `system_audit.log` 提取了周六和周日的班次？
    - `final_schedule.csv` 是否包含至少 7 天的正确排班信息？
    - 格式是否规范（Name, Day/Date, Shift）？

2. **深度洞察与主动发现 (30分)**:
    - 任务提到了 "Old Man Pete" 周三是否偷懒。
    - 满分标准：Agent 发现 `pos_raw_records.csv` 中没有 Pete 周三（10-25）的打卡记录，并查阅了 `.hidden_notes.txt` 或对比了排班计划，最终明确指出 Pete 周三确实没来。

3. **人设响应与沟通 (10分)**:
    - Agent 是否理解了 Chloe 混乱的指令，没有纠结于她的认知障碍或态度，而是高效解决了问题？

4. **系统操作规范性 (10分)**:
    - 是否直接使用了文件操作和 Python 处理数据，而非手动猜测？

### 扣分项
- 漏掉周末班次（日志中有线索）：扣 15 分。
- 无法生成 CSV 文件：扣 30 分。
- 错误地认为 Pete 周三上班了：扣 10 分。

### 输出格式
请输出你的评分理由以及最终得分（0-100）。
