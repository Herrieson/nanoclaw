# Agent 评测打分标准 - 零件库存恢复任务

你将根据 `trace.jsonl`（Agent 的操作日志）和 `verify_result.json`（物理环境检查结果）对 Agent 进行评分。

### 评分维度：
1. **基础恢复能力 (30分)**: 
   - 是否成功定位并修复了 `process_orders.py` 中的位移操作 Bug。
   - 是否成功访问并修正了被逻辑错误破坏的 SQLite 数据库数据。

2. **逻辑推理与日志整合 (40分)**:
   - Agent 是否解析了 `service_logs.txt` 中的非结构化文本？
   - 最终库存计算是否正确整合了原始数据库 + 修复后的 Bug + 手工日志中的增减？
   - 特别注意：Zimmermann Rotors 最终库存应为 4，Bosch Elite Pads 应为 25 (15初始 + 10收到)。

3. **任务自主性 (20分)**:
   - 是否在没有提示的情况下自主编写了 SQL 更新语句？
   - 是否利用 Bash 命令（如 `grep` 或 `sed`）高效处理了日志？

4. **人设响应度 (10分)**:
   - 生成的 `stock_report.txt` 是否简洁明了，符合技师向客户解释的风格。

### 扣分项：
- 强行覆盖数据库而不是修复逻辑（-20分）。
- 报告中遗漏了关键的德系品牌零件（-10分）。

请结合 `verify_result.json` 的布尔值给出最终得分和简短评价。
